1  Evaluating Diagnostic Tests with ROC Curves in R

Published

March 6, 2026

1.1 Introduction

Diagnostic tests in medicine and biology rarely produce a simple yes/no result. More often, they produce a continuous measurement, for example a biomarker concentration, an imaging score, a blood pressure reading. The researcher must decide how well that measurement distinguishes between individuals with and without a condition. Receiver Operating Characteristic (ROC) curves are one of the most widely used tools for evaluating this kind of diagnostic performance.

This guide walks through the full ROC analysis workflow in R: deciding whether ROC analysis is appropriate for your data, preparing your dataset, generating and visualizing the ROC curve, and computing and interpreting the Area Under the Curve (AUC). All steps are demonstrated with real datasets and annotated code using the pROC package (Robin et al. 2011).

This guide does not cover advanced topics such as covariate-adjusted ROC curves, time-dependent ROC analysis, or optimal threshold selection (e.g., the Youden Index). Those topics are excluded to keep the focus on foundational ROC analysis.

1.2 Prerequisites

1.2.1 Knowledge

You should be familiar with:

  • Sensitivity: the probability that the test is positive given the person has the disease (true positive rate)
  • Specificity: the probability that the test is negative given the person does not have the disease (true negative rate)
  • Binary outcomes: your reference standard should classify each individual as diseased (1) or non-diseased (0)

No prior experience with ROC curves is needed

1.4 Step 1: Is ROC Analysis Appropriate for Your Data?

Before writing any code, confirm that ROC analysis is the right tool.

ROC analysis is appropriate when:

  • Your diagnostic test produces a continuous or ordinal result (e.g., a blood glucose level, a probability score)
  • You have a binary reference standard — each observation is either diseased (1) or non-diseased (0)
  • Your goal is to evaluate overall discriminative ability across all possible thresholds, not performance at one specific cutoff

ROC analysis is not appropriate when:

  • Your test result is already binary (positive/negative)
  • Disease prevalence is very low
  • The costs of false positives and false negatives are very different — ROC curves treat both error types equally

1.4.1 Quick Checklist

Question Required Answer
Is the test result continuous or ordinal? Yes
Is the outcome binary (diseased / not diseased)? Yes
Am I interested in performance across all thresholds? Yes
Is disease prevalence reasonably balanced? Ideally yes

1.5 Step 2: Prepare Your Data

  • A numeric column for the test result

1.6 Step 3: Generate the ROC Curve

Use pROC::roc() to fit the ROC curve. The two key arguments are:

  • response: the binary outcome variable
  • predictor: the continuous test result
roc_asah <- roc(response = aSAH$outcome, predictor = aSAH$s100b)
Setting levels: control = Good, case = Poor
Setting direction: controls < cases
roc_asah

Call:
roc.default(response = aSAH$outcome, predictor = aSAH$s100b)

Data: aSAH$s100b in 72 controls (aSAH$outcome Good) < 41 cases (aSAH$outcome Poor).
Area under the curve: 0.7314

1.6.1 Understanding the Direction Warning

pROC will print a message about the direction it assumed (e.g., direction: controls < cases). This tells you whether higher test values are assumed to indicate disease. Do not ignore this. If the assumed direction is wrong for your test, your curve will be mirrored and your AUC will be below 0.5. You can set the direction explicitly:

# Higher s100b -> worse outcome: controls < cases (default is correct here)
roc_asah <- roc(response = aSAH$outcome,
                predictor = aSAH$s100b,
                direction = "<")
Setting levels: control = Good, case = Poor

1.7 Step 4: Compute and Interpret AUC

The Area Under the Curve (AUC) summarizes the ROC curve in a single number.

  • AUC = 0.5: the test performs no better than random chance
  • AUC = 1.0: the test perfectly separates diseased from non-diseased individuals
  • AUC = 0.7–0.8: generally considered acceptable discrimination
  • AUC > 0.8: generally considered excellent discrimination

These are rough benchmarks — what counts as “good enough” depends on your clinical or scientific context.

auc(roc_asah)
Area under the curve: 0.7314
ci.auc(roc_asah)
95% CI: 0.6301-0.8326 (DeLong)

1.8 Step 5: Visualize the ROC Curve

A good visualization shows the curve, the diagonal reference line (random chance), and the AUC value.

1.8.1 Basic Plot with pROC

plot(roc_asah,
     main = "ROC Curve: s100b predicting poor outcome (aSAH)",
     col = "#2c7bb6",
     lwd = 2)
legend("bottomright",
       legend = paste("AUC =", round(auc(roc_asah), 3)),
       col = "#2c7bb6", lwd = 2)

1.8.2 Polished Plot with ggplot2

roc_df <- data.frame(
  specificity = roc_asah$specificities,
  sensitivity = roc_asah$sensitivities
)

ggplot(roc_df, aes(x = 1 - specificity, y = sensitivity)) +
  geom_line(color = "#2c7bb6", linewidth = 1.2) +
  geom_abline(slope = 1, intercept = 0, linetype = "dashed", color = "gray50") +
  annotate("text", x = 0.75, y = 0.1,
           label = paste("AUC =", round(auc(roc_asah), 3)),
           size = 5, color = "#2c7bb6") +
  labs(title = "ROC Curve: s100b predicting poor outcome (aSAH)",
       x = "1 - Specificity (False Positive Rate)",
       y = "Sensitivity (True Positive Rate)") +
  theme_minimal()

1.9 Step 6: Evaluate Your Results

Once you have a curve and AUC, ask yourself:

  • Does the curve bow toward the upper-left corner? If it hugs the diagonal, the test has little discriminative ability.
  • Is the AUC above 0.5? If not, check whether the direction argument is set correctly.
  • Does the observation count in the pROC output match your dataset? If not, check for unexpected missing values.
  • Is the confidence interval for the AUC reasonably narrow? Wide intervals suggest insufficient sample size.
  • Does the AUC make sense given what is known about this test from prior literature?
